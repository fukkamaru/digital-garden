# ホスティングとは？

ホスティング（Hosting）は、  
「作ったWebサイトやデータを、インターネット上で公開・配信できる状態にすること」です。

もっと噛み砕くと、

- HTML
- CSS
- JavaScript
- 画像
- 動画
- データベース

などを、常時ネット接続されたサーバーに置き、  
他人がブラウザからアクセスできるようにする行為です。

たとえば：

- Quartzで生成した静的サイト
- WordPressブログ
- ECサイト
- 会社HP
- APIサーバー

などは、どこかのサーバー上に置かれています。

その「置き場所」と「配信機能」を提供するのがホスティングサービスです。

---

# ホスティングサイトとは？

「ホスティングサイト」は、  
ホスティングサービスを提供している事業者・サービスのことです。

例：

- GitHub の GitHub Pages
- Cloudflare の Cloudflare Pages
- Xserver
- Netlify

など。

---

# ホスティングの大分類

かなり重要です。

## ① 静的ホスティング（Static Hosting）

HTML/CSS/JSをそのまま配信。

特徴：

- 高速
- 安い
- セキュア
- Git連携が強い

Quartzと相性が良い。

代表：

- GitHub Pages
- Cloudflare Pages
- Netlify
- Vercel

---

## ② 動的ホスティング（Dynamic Hosting）

PHP・Node.js・Pythonなどが動く。

特徴：

- WordPress可
- DB利用可
- ログイン機能可
- 柔軟

代表：

- Xserver
- ConoHa
- Sakura
- Render
- Railway

---

## ③ VPS / クラウド

サーバーそのものを借りる。

特徴：

- 自由度最強
- 難易度高め
- Linux知識必要

代表：

- AWS
- GCP
- Azure
- さくらVPS

---

# GitHub Pagesとは？

GitHub が提供する静的ホスティング。

公式：  
[GitHub Pages](https://pages.github.com/?utm_source=chatgpt.com)

特徴：

- GitHubリポジトリから直接公開
- 無料
- Quartzと非常に相性が良い
- 技術者人気が高い
- 独自ドメイン可

ただし：

- 動的処理は基本不可
- サーバー側プログラム不可
- DBなし

つまり：

> 「静的サイト専用公開サービス」

です。

---

# Cloudflare Pagesとは？

Cloudflare が提供する静的ホスティング。

公式：  
[Cloudflare Pages](https://pages.cloudflare.com/?utm_source=chatgpt.com)

特徴：

- 超高速CDN
- 世界規模キャッシュ
- GitHub連携
- 独自ドメイン強い
- 無料枠強力
- Functionsで軽い動的処理可能

Quartz界隈でもかなり人気。

特に：

- GitHub Pagesより高機能
- CDN性能が強い
- DNS統合が便利

という理由で移行する人も多いです。

# Quartz用途ならどれが向いている？

|サービス|向いている人|

|GitHub Pages|シンプル重視|

- Quartz

- 独自ドメイン
- Zettelkasten/デジタルガーデン

> GitHub + Cloudflare Pages
