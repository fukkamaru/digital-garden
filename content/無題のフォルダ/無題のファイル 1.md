ざっくり言うと、こうです。

```
Git= 変更履歴を管理する道具GitHub= Gitで管理したデータを置くWebサービスGit LFS= Gitで扱いにくい大容量ファイルを別管理する仕組みGitHub Releases= 完成版・配布物を置く場所GitHub Pages= リポジトリの中身をWebサイトとして公開する機能
```

## それぞれの違い

|名称|何か|主な役割|例えるなら|
|---|---|---|---|
|**Git**|バージョン管理ツール|ファイルの変更履歴を記録する|作業履歴を残せる台帳|
|**GitHub**|Gitリポジトリを置くWebサービス|コードやファイルをオンラインで管理・共有する|Git用のクラウド保管庫|
|**Git LFS**|Gitの拡張機能|画像・動画・zipなど大容量ファイルをGit本体とは別に管理する|重い荷物だけ別倉庫に置く仕組み|
|**GitHub Releases**|GitHub上の配布機能|完成版のzip、exe、PDFなどを公開・配布する|製品版の配布棚|
|**GitHub Pages**|GitHubの静的サイト公開機能|HTML/CSS/JS/MarkdownなどをWebサイトとして公開する|GitHub上の無料Web公開スペース|

## 1. Git

**Gitは、ファイルの変更履歴を管理するための道具**です。

たとえばQuartzのファイルを編集したときに、

```
いつ誰がどのファイルをどう変更したか
```

を記録できます。

Gitで使う代表的な操作がこれです。

|操作|意味|
|---|---|
|`git init`|Git管理を始める|
|`git add`|記録対象に追加する|
|`git commit`|変更内容を履歴として保存する|
|`git push`|GitHubなどに送る|
|`git pull`|GitHubなどから取得する|
|`git clone`|リポジトリを丸ごとコピーする|

重要なのは、**GitそのものはローカルPCだけでも使える**ということです。

GitHubがなくてもGitは使えます。

## 2. GitHub

**GitHubは、Gitで管理したリポジトリをオンラインに置くサービス**です。

Gitが「道具」なら、GitHubは「置き場所」です。

```
Git= 変更履歴を管理する仕組みGitHub= そのGitリポジトリをネット上に置くサービス
```

GitHubを使うと、

```
複数端末で同期できるバックアップになる他人と共同作業できるWeb上でファイルを確認できるGitHub Pagesで公開できるCloudflare Pagesと連携できる
```

といったメリットがあります。

Quartzの場合は、

```
Obsidianで編集↓Gitで変更履歴を記録↓GitHubにpush↓Cloudflare PagesやGitHub Pagesで公開
```

という流れになります。

## 3. Git LFS

**Git LFSは、大容量ファイルをGitで扱いやすくするための拡張機能**です。

LFSは **Large File Storage** の略です。

通常のGitは、テキストファイルの履歴管理が得意です。Markdown、HTML、CSS、JavaScript、設定ファイルなどですね。

一方で、次のようなファイルは苦手です。

```
動画高解像度画像PSDAIデータzipPDF音声ファイル
```

こういう大きいファイルを普通にGitへ入れると、リポジトリがどんどん重くなります。

そこでGit LFSを使うと、Git本体には実ファイルではなく、軽いポインタファイルだけを置きます。

```
Git本体= この大きいファイルはLFS側にあるよ、という情報だけ持つGit LFS= 実際の大容量ファイルを保管する
```

ただし、重要な注意点があります。

**GitHub PagesではGit LFSのファイルをそのままサイト公開素材として使う用途には向きません。**

なので、Quartzの画像やPDFをWeb公開したい場合は、

```
小さい画像・PDF→ 通常のGit管理大きいPDF・動画・巨大画像→ Cloudflare R2、Google Drive、YouTube、別ストレージなど
```

のほうが実務上は安全です。

## 4. GitHub Releases

**GitHub Releasesは、完成版のファイルを配布するための場所**です。

たとえばソフトウェア開発なら、

```
v1.0.0v1.1.0v2.0.0
```

のようにバージョンごとに公開できます。

そこに、

```
zipexedmgpdfマニュアルソースコード一式配布用データ
```

などを添付できます。

GitHub Releasesは、通常のWebサイト素材置き場というより、

```
完成版を配る場所
```

です。

たとえば、

```
Quartzサイトのソース→ GitHubリポジトリWebサイトとして公開→ GitHub Pages / Cloudflare Pages配布用zipやPDF→ GitHub Releases
```

という使い分けになります。

## 5. GitHub Pages

**GitHub Pagesは、GitHubリポジトリの中身をWebサイトとして公開する機能**です。

公開できるのは、基本的に静的サイトです。

```
HTMLCSSJavaScript画像Markdownから生成されたHTML
```

などです。

WordPressのようにサーバー側でPHPを動かしたり、データベースを使ったりするものではありません。

Quartzは静的サイトジェネレーターなので、GitHub Pagesと相性が良いです。

```
ObsidianのMarkdown↓QuartzでHTMLに変換↓GitHub Pagesで公開
```

ということができます。

ただし、GitHub Pagesは「大容量ファイル置き場」ではありません。サイト全体サイズや帯域幅の制限があるため、PDF・動画・高解像度画像を大量に置く用途には向きません。

## 関係図

```
Git└─ ローカルで変更履歴を管理するGitHub└─ Gitリポジトリをネット上に置くGit LFS└─ GitHub上で大容量ファイルを別管理するGitHub Releases└─ 完成版ファイルを配布するGitHub Pages└─ リポジトリの内容をWebサイトとして公開する
```

もう少しQuartz寄りにすると、こうです。

```
Obsidianでノートを書く↓Gitで履歴管理する↓GitHubにpushする↓GitHub PagesまたはCloudflare Pagesで公開する
```

大きいファイルがある場合は、

```
大きい画像・PDF・動画↓Git LFS、GitHub Releases、R2などを検討
```

ただしWebサイト内で安定して表示・配信したいなら、Git LFSやReleasesより、**Cloudflare R2などのストレージに逃がすほうが自然**です。

## 一言でまとめると

|名前|一言でいうと|
|---|---|
|Git|履歴管理する道具|
|GitHub|Gitの置き場・共有サービス|
|Git LFS|大容量ファイルをGitとは別に扱う仕組み|
|GitHub Releases|完成版・配布物を置く場所|
|GitHub Pages|GitHubからWebサイトを公開する機能|

特に混同しやすいのは、**GitHub ReleasesとGitHub Pages**です。

```
GitHub Releases= ファイルを配る場所GitHub Pages= Webサイトを見せる場所
```

この区別で見ると、かなり整理しやすいです。
