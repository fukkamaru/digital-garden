## 最終テンプレ（これで固定してOK）

```
{product}-{usage}-{background}-{size}.{ext}
```

### 実例

```
kanpate-epoxy-thumb-transparent-300x300.png
kanpate-epoxy-main-white-1200x1200.jpg
```

---

### 一言まとめ

- **背景色（透過含む）はサイズより前**
- **用途の直後が最も安定**
- **サイズは常に一番お尻**


ホームページに画像をアップロードすると、プラグインにより異なるサイズをいくつか生成します。 そのため、サイズ被りが起こると、最後に−01や−02画付いたりします。 気にしないで放っておくか、サイズ表記をres280x280みたいに、頭にｒｅｓを付けるのと、どちらが良い？

表示を増やす表示を減らす

結論を先に言います。

**👉「気にしないで放っておく」が正解。  
`res280x280` のような表記は付けないほうが良い。**

---

## 理由①：それは「管理すべき画像」ではない

WordPress等のプラグインが自動生成する

```
product-thumb-280x280.jpgproduct-thumb-280x280-01.jpgproduct-thumb-280x280-02.jpg
```

これらは **CMS内部都合の派生物** です。

- 人が選んで使う画像 → **元画像**
- 表示最適化のための自動生成 → **システム資産**

👉 **命名の美しさや一貫性を、人間側で担保する対象ではない**


## 正しい割り切り方（重要）

### ① 人が管理するのは「元画像だけ」

```
product-thumb-transparent.pngproduct-main-white.jpg
```

※ **サイズを書かないのが最も安全**

---

### ② サイズ違いは「CMSの責任」に切り出す

- `-280x280`
- `-300x300`
- `-01`

👉 **全部ノイズとして無視**

消える・増える・変わる前提でOK。
