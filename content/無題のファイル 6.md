マークダウンにおけるコールアウトの使い分けをもう一回教えて

Markdownのコールアウトは、ざっくり言うと **「本文とは別枠で、読み手に扱い方を変えてほしい情報」** を入れる箱です。

通常本文で流せるなら本文でよく、コールアウトにするのは **注意・補足・要点・例外・引用・作業メモ** など、情報の役割を明確にしたいときです。

## 基本の使い分け

### note：普通の補足

一番よく使う汎用枠です。

```
> [!note]> ここでは、本山という語を宗派の中心寺院という広い意味で使う。
```

使う場面は、本文の理解を助ける補足、定義のゆるい確認、前提条件などです。

迷ったら `note` でOKです。

---

### info：案内・背景情報

`note` より少し「案内」っぽい情報に向きます。

```
> [!info]> 宗派によって、本山・大本山・総本山の使い方は異なる。
```

「読者に知っておいてほしい背景」「制度や用語の前提」を入れるときに使いやすいです。

---

### tip：実用的なコツ

読者がすぐ使える判断基準や運用ルールに向きます。

```
> [!tip]> 迷った場合は、まず「総本山 → 大本山 → 本山」の順で整理し、宗派ごとの例外は後から補足すると見やすい。
```

ノート作成、Notion運用、Markdown運用、Excel操作などの「実務上のコツ」に相性がいいです。

---

### warning：注意・誤解防止

間違いやすい点、例外、読み違いを防ぎたいときに使います。

```
> [!warning]> 「本山」は必ずしも一宗派に一つだけとは限らない。宗派や流派によって複数存在する場合がある。
```

強い危険というより、**読み手に立ち止まってほしい情報** です。

---

### caution / danger：強い注意・リスク

`warning` より重いです。

```
> [!danger]> 公式情報を確認せずに寺格を断定すると、宗派ごとの実態とズレる可能性がある。
```

ただし、歴史・宗教・読書ノート系では使いすぎると大げさです。  
基本は `warning` までで十分です。

---

### example：具体例

本文で説明した概念に、例を添えるときに使います。

```
> [!example]> 例：天台宗では比叡山延暦寺が総本山として扱われる。
```

「説明」と「例」を分けたいときに便利です。

---

### quote：引用・原文メモ

引用文や、あとで出典確認したい文章に使います。

```
> [!quote]> 本文から引用した文章をここに入れる。
```

自分の解釈ではなく、外部資料・本・公式文から取った内容だと分かるようにできます。

---

### question：疑問・未解決メモ

あとで調べたいこと、まだ結論が出ていない論点に使います。

```
> [!question]> 「大本山」と「別格本山」は、宗派ごとにどのように使い分けられているのか？
```

ZKや読書ノートではかなり便利です。  
「今は断定しない」情報を分離できます。

---

### summary / abstract：要約

章やノートの冒頭・末尾に要点を置くときに使います。

```
> [!summary]> 本山とは、宗派や流派の中心となる寺院を指す。ただし、総本山・大本山・本山の序列や意味は宗派によって異なる。
```

長めのノートでは、最初に置くと再読しやすくなります。

---

## あなたのノート運用なら、まずこの6種類で十分

使い分けを増やしすぎると面倒になるので、基本はこれで足ります。

|種類|用途|
|---|---|
|`note`|普通の補足|
|`info`|背景・前提|
|`tip`|実務上のコツ・判断基準|
|`warning`|誤解防止・注意点|
|`example`|具体例|
|`question`|未解決・あとで調べる|

かなり現実的には、**note / warning / example / question** の4つだけでも回ります。

## 判断基準

本文に入れるべきものは、**説明の本筋**です。

コールアウトに入れるべきものは、**本筋から少し外れるけれど、読者の理解や運用に役立つもの**です。

たとえば「本山」のノートなら、

```
## 本山とは本山とは、宗派や流派の中心となる寺院を指す言葉である。ただし、宗派によって総本山・大本山・本山などの使い方は異なる。> [!warning]> これらの呼称は、すべての宗派で統一された階層制度ではない。## 序列の目安1. 総本山2. 大本山3. 本山> [!note]> 実際には「別格本山」「準別格本山」「門跡寺院」など、宗派ごとの独自の格付けが存在する場合もある。
```

このような感じです。

## 結論

コールアウトは、**装飾ではなく情報の役割ラベル**として使うのが一番いいです。

特にあなたのZK・Quartz運用なら、最初はこうで十分です。

```
> [!note] 補足> 本文の理解を助ける情報。> [!warning] 注意> 誤解しやすい点、例外、断定を避けたい点。> [!example] 例> 具体例。> [!question] 未解決> あとで調べたいこと。
```

細かく使い分けるより、**「本文」「補足」「注意」「例」「未解決」くらいに分ける**のが続けやすいです。
